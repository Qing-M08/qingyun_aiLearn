## 一、项目概述

**青云智学**是一个 AI 驱动的全流程学习辅助软件后端系统，采用 FastAPI + SQLAlchemy + Celery + PostgreSQL 技术栈，提供智能学习路线生成、苏格拉底式答疑、间隔重复复习等核心功能。

### 1.1 技术栈

| 组件 | 技术选型 | 版本 |
|------|---------|------|
| Web 框架 | FastAPI | ≥0.115 |
| ORM | SQLAlchemy | ≥2.0 (async) |
| 数据库 | PostgreSQL | 16 |
| 图数据库扩展 | Apache AGE | - |
| 向量检索 | pgvector | ≥0.3 |
| 缓存/消息队列 | Redis | 7 |
| 异步任务 | Celery | ≥5.4 |
| 搜索引擎 | Meilisearch | v1.8 |
| LLM 服务 | DeepSeek API | OpenAI 兼容接口 |
| 嵌入模型 | sentence-transformers | bge-m3 |
| 认证 | JWT | python-jose |
| 日志 | structlog | ≥24.1 |

### 1.2 基础设施

- **容器化**：Docker Compose（PostgreSQL + Redis + Meilisearch）
- **数据库迁移**：Alembic
- **代码规范**：Ruff + MyPy
- **Python 版本**：3.11+

### 2.2 核心架构特性

1. **全异步架构**：FastAPI + SQLAlchemy 2.0 async + asyncpg
2. **依赖注入**：FastAPI Depends 机制管理数据库会话和用户认证
3. **分层设计**：API 路由 → Service 服务 → ORM 模型
4. **异步任务**：Celery + Redis 处理 LLM 调用、路线生成等耗时操作
5. **WebSocket 实时通信**：支持讲义进度、答疑流式输出、复习提醒
6. **图数据库集成**：Apache AGE 扩展支持知识图谱查询
7. **RAG 支持**：向量嵌入 + 文档分块 + 检索增强生成

#### Sprint 7：复习系统 ✅

**交付内容**：
- ✅ SM-2 间隔重复算法
- ✅ 复习计划管理（创建/列表/完成）
- ✅ Celery Beat 定时提醒（每 5 分钟检查到期复习）
- ✅ 复习内容自动生成（根据掌握度选择闪卡/选择题/讲解）

**核心实现**：

**1. 复习服务（`review_service.py`）**：
- SM-2 算法：
  - `calculate_next_review` - 核心算法
    - quality >= 3: interval 递增（0→1→6→round(interval*ef)），ef 更新
    - quality < 3: interval 重置为 1，ef 降低 0.2（最低 1.3）
  - `quality_from_performance` - 0-1 分数转 0-5 质量评分
- 复习计划管理：
  - `schedule_review` - 创建/更新复习计划
  - `get_review_plans` - 列表（含节点名称，支持 status/日期范围筛选+分页）
  - `complete_review` - 完成复习：标记 completed → SM-2 计算 → 更新掌握度 → 自动创建下次计划
- 复习统计：今日到期、本周完成、逾期数、掌握度分布
- 复习内容生成：
  - 根据掌握度自动选择类型：>0.7→flashcard, 0.4-0.7→quiz, <0.4→explanation

## 五、核心算法与实现细节

### 5.1 知识掌握度更新（指数移动平均）

```python
# 正确回答
mastery_score += (1 - mastery_score) * 0.3

# 错误回答
mastery_score *= 0.7
```

### 5.2 SM-2 间隔重复算法

```python
# quality >= 3（回答正确）
if interval == 0:
    interval = 1
elif interval == 1:
    interval = 6
else:
    interval = round(interval * ease_factor)

ease_factor = ease_factor + (0.1 - (5 - quality) * (0.08 + (5 - quality) * 0.02))
ease_factor = max(1.3, ease_factor)

# quality < 3（回答错误）
interval = 1
ease_factor = max(1.3, ease_factor - 0.2)

next_review = now + timedelta(days=interval)
```

### 5.3 学习路线动态调整

**表现不佳**（用时/预估 > 2）：
- 添加补充练习步骤
- 后续步骤顺延

**表现优秀**（用时/预估 < 0.5）：
- 合并后续两个步骤
- 跳过第二个步骤

### 5.4 复习内容自动选择

```python
if mastery_score > 0.7:
    review_type = "flashcard"      # 闪卡
elif mastery_score > 0.4:
    review_type = "quiz"           # 选择题/填空题
else:
    review_type = "explanation"    # 重新讲解
```

### 8.1 ConnectionManager

class ConnectionManager:
    active_connections: dict[str, WebSocket]  # client_id -> websocket

### 8.2 Redis Pub/Sub 跨进程通信（Sprint 8）

讲义生成进度通过 Redis Pub/Sub 实现 Celery worker → FastAPI 跨进程推送：

```python
# 发布（Celery worker 进程，同步 Redis 客户端避免 event loop 冲突）
def publish_lecture_progress(lecture_id: str, data: dict):
    _sync_redis.publish(f"lecture_progress:{lecture_id}", json.dumps(data))

# 订阅（FastAPI WebSocket 进程，async Redis 客户端）
async_redis = redis.Redis.from_url("redis://localhost:6379/3")
pubsub = async_redis.pubsub()
await pubsub.subscribe(f"lecture_progress:{lecture_id}")
```

**进度阶段**：
| 阶段 | 百分比 | 说明 |
|------|--------|------|
| connecting | 0% | WebSocket 连接建立（FastAPI 发送） |
| preparing_context | 10% | 任务启动，查询知识图谱/RAG/网络搜索 |
| generating_content | 30% | 调用 LLM 生成讲义内容 |
| finalizing | 80% | LLM 返回，保存讲义 + 创建关联笔记 |
| complete | 100% | 全部完成，携带 lecture 数据 + note_id |
| error | - | 异常时推送错误消息 |

**设计要点**：
- 发布端使用**同步 Redis**（`redis.Redis`）避免 Celery worker 中的 event loop 冲突。
  Celery `run_async()` 每次任务创建新 loop 并在完成后关闭，`redis.asyncio` 缓存连接会绑定到已关闭的 loop 导致 `RuntimeError: Event loop is closed`。
- 订阅端每次 WebSocket 连接创建**新的 async Redis 客户端**，确保连接绑定到正确的 event loop。
